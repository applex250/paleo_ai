# 工作清单（work_list）

> 评估基准：以「是否具备真实可用逻辑」为准。能跑、能看、能点的 UI 算完成；按钮/数据是写死的、靠 Mock、点了没反应的算待完成。
>
> **一句话总览**：8 个功能模块的**界面（UI）已全部完成**，但本质上是一个**纯前端原型**——几乎没有后端、没有数据持久化，除「标注画布」外的大多数交互是装饰性按钮或写死数据。真正可交互的核心只有标注工作台。

## 三、⏳ 待完成的工作

### A. 后端与数据持久化（最大缺口）
- [ ] **没有任何后端服务**，全部数据为前端 Mock
- [ ] 标注结果**无法保存**：「提交保存」按钮无逻辑，刷新即丢失
- [ ] 无数据库 / 数据集真实存储；`mockDatasets` 等是写死数组
- [ ] 无用户、任务、模型的增删改查接口

### B. 各页面的「假交互」需要补真实逻辑
- [ ] **DataManager**：「连接海油数据湖 / S3 / PostgreSQL」按钮无逻辑；「数据清洗」Tab 是占位（清洗向导未实现）；「特征构造」三张卡片无实际处理
- [ ] **Annotation**：「提交保存」无保存逻辑；「智能辅助建议 / 一键应用预标注」是**写死的占位文本**（无真实 AI 推理）；仅支持本地上传，无图片库/任务分发/审核流
- [ ] **Training**：「提交任务」无逻辑；任务的**启动/暂停/终止按钮是装饰性的**（无 handler）；无真实训练调度、无 Loss 曲线实时更新
- [ ] **Evaluation**：版本对比下拉框无切换效果；指标数据写死
- [ ] **ModelRegistry**：纯展示，无部署 / 下线 / 归档 / 版本切换操作
- [ ] **Serving**：推理是 `setTimeout` 800ms 后**返回写死的 JSON**，无真实模型服务、无真实 QPS/延迟
- [ ] **Monitoring**：图表数据用 `Math.random` 现场生成，日志流是写死的 HTML，无真实指标采集
- [ ] **Dashboard**：4 个指标数值写死（"12" / "45 TB" / "87%" / "8"）

### C. 全局 / 通用功能缺口
- [ ] 无登录鉴权（顶部「管理员 / 平台工程师」是静态文本）
- [ ] 顶部搜索框无功能
- [ ] 通知铃铛无功能
- [ ] `Sidebar`「系统设置」按钮无功能
- [ ] 无跨页面状态管理（未引入 Redux / Context，数据无法共享）
- [ ] 无单元测试 / E2E 测试
- [ ] 无统一 loading / 错误态 / 错误边界处理
- [ ] 无权限角色（标注员 / 审核员 / 管理员）区分

### D. 工程清理与小修
- [ ] `App.tsx` 中的 `Placeholder` 组件已是**死代码**（所有路由都接了真实页面），可删除
- [ ] `index.html` 的 importmap（React 19）与 `package.json`（React 18）**版本不一致**，建议统一
- [ ] Tailwind 走 CDN，生产环境建议改为本地构建链路（PostCSS）
- [ ] 原 `README.md` 提到的 `GEMINI_API_KEY` / `.env.local` 实际代码未使用，属 AI Studio 模板残留，可清理

## 四、建议优先级

| 优先级 | 工作项 | 理由 |
|:---:|------|------|
| **P0** | 标注结果保存 + 最小后端（数据集/标注 CRUD） | 标注是核心功能，目前做完即丢，无法形成闭环 |
| **P0** | 标注「提交保存」「智能辅助预标注」接真实逻辑 | 核心交互目前是假的 |
| **P1** | 登录鉴权 + 角色权限 | 多人协作前提 |
| **P1** | Training 任务提交/启停接真实调度（哪怕是假后端） | 让训练页能用 |
| **P1** | Serving 推理接真实模型服务 | 让推理页能用 |
| **P2** | Monitoring 接真实指标采集、Dashboard 指标取真实数据 | 运营监控价值 |
| **P2** | 死代码清理、版本统一、Tailwind 本地化 | 工程质量 |
